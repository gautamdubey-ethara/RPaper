# An Empirical Comparison of Parameter-Efficient Mitigation Strategies for Catastrophic Forgetting in Small LLMs under Compute-Constrained Continual Fine-Tuning

**Author:** Gautam Dubey (gautamdubey296@gmail.com)

## Overview

This repository contains the code, data loaders, and LaTeX source for the paper
*An Empirical Comparison of Parameter-Efficient Mitigation Strategies for Catastrophic Forgetting
in Small LLMs under Compute-Constrained Continual Fine-Tuning*.

We benchmark six continual fine-tuning methods on a four-task GLUE sequence
(SST-2 → MRPC → CoLA → MNLI) using QLoRA on two small instruction-tuned LLMs
(Qwen2.5-1.5B-Instruct and Phi-3.5-mini-instruct) on a single NVIDIA T4 GPU.

**Methods evaluated:**

| ID | Method |
|----|--------|
| `full_ft` | Full Fine-Tuning |
| `lora` | Vanilla LoRA |
| `lora_ewc` | LoRA + EWC |
| `lora_replay` | LoRA + Experience Replay |
| `olora` | O-LoRA |
| `mmr` | Minimal Mixed Rehearsal (**ours**) |

## Quickstart (Google Colab)

1. Open `code/colab_setup.ipynb` in Google Colab.
2. Set runtime to **GPU (T4)**: Runtime → Change runtime type → T4 GPU.
3. Run all cells in order.

Cell 4 runs the full sweep for Qwen2.5-1.5B-Instruct with all six methods (seed 42).
Cell 5 generates the three paper figures and saves them to `paper/figures/`.

## Local Reproduction

```bash
git clone https://github.com/gautamdubey-ethara/RPaper.git
cd RPaper/code
pip install -r requirements.txt

# Run all methods for Qwen, seed 42
python run_experiments.py \
    --model qwen \
    --methods full_ft,lora,lora_ewc,lora_replay,olora,mmr \
    --seeds 42 \
    --output_dir ../results \
    --n_train 2000 \
    --epochs 2

# Run Phi-3.5 main comparison
python run_experiments.py \
    --model phi \
    --methods full_ft,lora,lora_ewc,lora_replay,olora,mmr \
    --seeds 42 \
    --output_dir ../results
```

Results are saved as `results/{model}_{method}_{seed}.json`.

## Compiling the Paper

Requires a LaTeX distribution with IEEEtran, `algorithm`, `algorithmic`, `booktabs`,
and `graphicx` packages (standard in TeX Live / MiKTeX).

```bash
cd paper
pdflatex main.tex
bibtex main
pdflatex main.tex
pdflatex main.tex
```

The compiled PDF will be `paper/main.pdf`.

## Results Structure

Each `results/{model}_{method}_{seed}.json` file contains:

```json
{
  "model": "qwen",
  "method": "mmr",
  "seed": 42,
  "R_matrix": [[...], ...],
  "average_accuracy": 0.0,
  "forgetting": 0.0,
  "backward_transfer": 0.0,
  "timing_per_task": {"sst2": 0.0, ...},
  "memory_per_task": {"sst2": 0.0, ...},
  "total_time_s": 0.0,
  "peak_mem_gb": 0.0,
  "hparams": {...}
}
```

`R_matrix[i][j]` is the evaluation metric on task `j` after training through task `i`.
